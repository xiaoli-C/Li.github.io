bash 基本功能

1.1.history-历史命令

  • 语法：history [选项] [历史命令保存文件]

  • history -c ~/.bash_history : 清除历史命令

  • history -w /root/.bash_history : 将缓存中的命令写入保存文件 ~/.bash_history(默认历史命令保存文件)

  • 默认保存1000条 ；在/etc/profile中的HISTSIZE值进行修改

  • 历史命令的调用

    • 上下箭头
    • !n：执行第n条历史命令
    • !!：执行上一条命令
    • !string：执行最后一次执行的包含string的命令

1.2.tab键

  • 补全功能

1.3.alias-别名

  • alias 别名=’原命令’

    alias ls=’ls –color=auto’

  • alias 查询命令别名

  • 命令执行顺序

    • 第一先执行绝对路径或相对路径的命令，~/bin/ls
    • 第二执行别名
    • 第三执行Bash内部命令，cd
    • 第四执行$PATH环境变量找到的第一个命令

  • 别名永久生效

    • /root/.bashrc

  • 删除别名

    • unalias 别名

1.5. 输入输出重定向

  • 标准输出重定向

    • 命令 > 文件

      将命令输入到指定文件中，如果文件不存在则创建文件，输入内容将文件原内容覆盖。

    • 命令 >> 文件

      将命令输入到指定文件中，如果文件不存在则创建文件，输入内容将输入到文件末尾；

    • 如果操作失误将两个>输成一个，那将会覆盖不需覆盖的文件

      • set -C 禁止覆盖命令

      • set +C 解除覆盖命令

        [root@localhost ~]# set -C 
[root@localhost ~]# echo 789 > a.txt
-bash: a.txt: 无法覆盖已存在的文件
[root@localhost ~]# set +C
  • 标准错误输出重定向
    • 错误命令 2> 文件
    • 错误命令 2>> 文件

1.6.多命令执行顺序

  • ;

    • 命令1;命令2；命令3
    • 命令顺序执行
    • 命令出现错误互不影响

  • && ：

    • 命令1;命令2；命令3
    • 命令1正确命令2才能执行

  • ||：

    • 命令1;命令2；命令3
    • 命令1执行，命令2不执行
    • 命令1不执行，命令2执行

1.7.管道符|

  • 命令1 | 命令2

  • 命令1的执行结果作为命令2的操作对象

  • 和grep使用举例

    [root@localhost ~]# ifconfig | grep netmask  
        inet 192.168.20.128  netmask 255.255.255.0  broadcast 192.168.20.255
        inet 127.0.0.1  netmask 255.0.0.0
//ifconfig的内容作为grep命令的操作对象
