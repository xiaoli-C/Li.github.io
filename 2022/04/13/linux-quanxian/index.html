linux-权限

linux
发布日期:   2022-04-13

ACL权限:访问控制列表

查看ACL命令

  • getfacl 文件名：查看ACL权限

    getfacl a.txt

    #file: a.txt
#owner: root
#group: root
user::rw-
user:zhr:r-x
group::r--
mask::r-x
other::r--

设定ACL命令

  • setfacl [选项] [文件名]
    • -m：设定ACL权限
    • -x：删除指定的ACL权限
    • -b：删除所有的ACL权限
    • d：设定默认ACL权限
    • -k：删除默认ACL权限
    • -R：递归设定ACL权限

给用户和用户组添加ACL权限

  • setfacl -m u:user_name:rx /project
    • 以 u:需要分配权限的用户名:权限 的格式分配
  • setfacl -m g:group_name:rx /project

最大有效权限值

  • mask:rwx
    • 我们赋予用户的ACL权限需要与mask权限”相与”得到用户真正的权限
    • r | r = r
    • r | - = -
    • -| - = -
  • 修改最大有效权限
    • setfacl -m m:rx 文件名

删除ACL权限

  • setfacl -x u:用户名 文件名
    • 删除用户的ACL权限
  • setfacl -x g:组名 文件名1
    • 删除用户组的ACL权限
  • setfacl -b 文件名
    • 删除文件的所有ACL权限

递归ACL权限

  • 给父目录设定ACL权限时，其所有子目录及子文件拥有相同的ACL权限
  • setfacl -m u:user_name:rx -R 文件名

默认ACL权限

  • 默认ACL权限的作用是给父目录设定默认的ACL权限，父目录中新建的子文件都会继承父目录的ACL权限，原来的文件不受影响。

  • setfacl -m d:u:用户名:权限 目录名

    [root@localhost ~]# setfacl -m d:u:zhr:rw japan  //设置zhr对目录Japan的默认权限
[root@localhost ~]# setfacl -m d:u:chy:w japan  //设置chy对目录Japan的默认权限
[root@localhost ~]# mkdir japan/cjk  //新建文件夹
[root@localhost ~]# touch japan/cls.av //新建文件
[root@localhost ~]# getfacl japan/cjk //查看新建的文件ACL权限
# file: japan/cjk
# owner: root
# group: root
user::rwx
user:zhr:rw-  
user:chy:-w-
group::r-x
mask::rwx
other::r-x
default:user::rwx
default:user:zhr:rw- //继承默认权限
default:user:chy:-w- 
default:group::r-x
default:mask::rwx
default:other::r-x
[root@localhost ~]# getfacl japan/cls.av
# file: japan/cls.av
# owner: root
# group: root
user::rw-
user:zhr:rw- //继承权限
user:chy:-w- //继承权限
group::r-x			
mask::rw-
other::r--

文件特殊权限

SetUID

  • 只有可以执行的二进制程序才能设定SUID权限

  • 命令执行者要对该程序拥有x执行权限

  • 命令执行者在执行该程序时获得该程序文件属主的身份

  • SetUID权限只在该程序执行过程中有效

  • 例如修改密码要修改/etc/shadow,但普通用户没有权限，但/usr/bin/passwd拥有SUID权限，当普通用户执行passwd命令时普通用户获得root身份。

  • /usr/bin/passwd权限 ：-rwsr-xr-x

  • 设置SUID

    • chmod 4755 文件名
    • chmod u+s 文件名

  • 取消

    • chmod 755 文件名 重新指回来
    • chmod u-s 文件名

SetGID

  • 针对文件的作用

    • 只有可执行的二进制程序才能设置SGID权限
    • 命令执行者对该程序有x权限
    • 命令执行者在执行程序的时候，组身份升级为该二进制文件的所属组
    • SGID只在程序执行过程中有效
    • 示例：普通用户执行locate命令需要查询mlocate.db,但普通用户对mloacate.db无任何权限，但locate有SGID权限，普通用户在执行这个命令的时候会获得组身份。

  • 针对目录的作用

    • 普通用户必须对此目录拥有r和x权限，才能进入该目录
    • 普通用户在此目录中的有效组会变成此目录的属组
    • 若普通用户对此目录拥有w权限时，新建的文件的默认数组是这个目录的数组

  • 设置

    • chmod 2755 文件
    • chmod g+s 文件

  • 取消

    • chmod 755 文件
    • chmod g-s 文件

Sticky BIT

  • 作用

    • 只对目录有效
    • 普通用户对目录有w和x权限
    • 对目录赋予SBIT，除了root可以删除所有文件，普通用户就算有w权限，也只可以删除自己创建的文件，不能删除其他用户的文件

  • 设置SBIT

    • chmod 1755 目录名
    • chmod o+t 目录名

  • 取消SBIT

    • chmod 755 目录名
    • chmod o-t 目录名

chattr 文件系统属性权限

  • 命令格式
    • chattr [+-=] [选项] 文件或目录名
    • +：增加；-：删除；=：等于；
  • 选项
    • i：对文件设置i属性，即不可对文件删除改名不能添加和修改数据；对目录设置，只能修改目录下的文件的数据，但不允许建立和删除文件。
    • a：对文件只能增加数据，不能删除和修改数据；对目录设置，只允许在目录中建立和修改文件，不允许删除。

查看文件系统属性

  • lsattr [选项] 文件名
    • -a : 显示所有文件和目录
    • -d : 若目标是目录，仅列出目录本身的属性，而不是子文件的

sudo权限

  • root将一些只有root用户执行的命令赋予给普通用户执行

  • visudo-赋予用户权限命令

    • 实际修改的是 /etc/sudoers 文件
    • /etc/sudoers文件不能用vim修改，只能用visudo命令修改
    • visudo是修改这个文件的专属命令

  • 文件修改规则

    root ALL=(ALL) ALL

    被授权用户 被管理主机的地址 授权命令

    %wheel ALL=(ALL) ALL

    %组名 被管理主机的地址=(可使用的身份) 授权命令

    zhr ALL=(ALL) /sbin/shutdown -r now

  • 普通用户执行

    • sudo -l：查看可以执行的sudo命令 
    用户 zhr 可以在 localhost 上运行以下命令：
    (ALL) /sbin/shutdown -r now
    • 命令执行
      • sudo /sbin/shutdown -r now
